# Privacy

This document is normative for CodexJournal-Lite. If a future change weakens
any guarantee below, update this file in the same change.

## What The Project Reads

CodexJournal-Lite reads session files from multiple AI coding assistants.
Each source has its own configured local directory:

- **Codex:** `%USERPROFILE%\.codex\sessions` (default, configurable via
  `config.json -> sessionsDir` or `--sessions-dir`)
- **Claude Code:** `~/.claude/projects` (auto-detected cross-platform)
- **Gemini CLI:** `~/.gemini/tmp` (auto-detected, disabled by default)
- **OpenCode:** `~/.local/share/opencode` (Unix) / `%LOCALAPPDATA%\opencode`
  (Windows), disabled by default
- **JetBrains (IDEA):** read-only inventory probe only

Supported file extensions vary by source: `.jsonl` and `.transcript`
(Codex), `.jsonl` (Claude Code), `.json` (Gemini CLI, OpenCode file mode).

The archive reader walks each source directory and parses session files.
It does not modify or delete the original session files.

The optional IDEA / JetBrains source scan is inventory-only. It looks for
candidate log files and writes a redacted inventory report. It does not import
IDEA log content into the journal or task archive.

## What The Project Writes

The project writes only inside its own project root.

| Path | Audience | Sanitized | Gitignored |
| --- | --- | --- | --- |
| `reports/dashboard.md` | local user | yes | yes |
| `journal/*.md` | local user | yes | yes |
| `data/tasks.json` | local user | yes | yes |
| `data/stats.json` | local user | yes | yes |
| `data/search.md` | local user | yes | yes |
| `data/patterns.json` | local user | yes | yes |

| `reports/*.md` / `reports/*.json` | local user | yes | yes |
| `dist/*.zip` | local handoff | yes, by included outputs | yes |

The public `README.md` is documentation and is not regenerated by
`npm run archive`.

## Network Behaviour

- Archive, analysis, source-scan, doctor, package, and verification commands do
  not make network calls.
- The OpenCode adapter in `mode: "cli"` spawns the local `opencode` binary
  (`opencode session list` / `opencode session export`) via `child_process`.
  This is a local subprocess call, not a network request. No data is uploaded.
- The dashboard binds to `127.0.0.1` by default.
- No telemetry, analytics beacon, error reporting service, or upload endpoint is
  included.
- `package.json` declares no runtime dependencies.

## Redaction Coverage

`src/sanitize.js` runs on strings that end up in generated user-facing archive
outputs, including `journal/*.md`, `data/tasks.json`, `data/search.md`,
`reports/dashboard.md`, report files, and the embedded task fields inside
`data/index.json`.

The redaction layer is covered by offline regression tests in
`src/sanitize.test.js` (`npm run test:sanitize`). These tests confirm that
common token, key, cookie, and path patterns are detected and replaced, without
requiring real data or network access.

Patterns currently redacted include:

- OpenAI-style API keys such as `sk-...` and `sk-proj-...`
- Generic API key assignments
- Common AI provider env-var assignments
- Authorization bearer values
- Cookie and session identifiers
- Access and refresh tokens
- GitHub personal access token shapes
- Slack token shapes
- Windows user paths under `C:\Users\<name>`
- Linux home paths under `/home/<name>/`
- macOS home paths under `/Users/<name>/`

The local user names recognised by the redaction layer come from
`os.userInfo().username`, `USERNAME`, `USER`, and the basename of the home
directory. Generic names such as `public`, `administrator`, `default`, and
`user` are ignored so unrelated text is not rewritten.

### Custom redactPatterns

Users can define additional redaction rules in `config.json -> redactPatterns[]`.
Each entry supports the following fields:

| Field | Required | Description |
| --- | --- | --- |
| `pattern` | yes | A regular expression string to match |
| `replacement` | yes | The string to replace matched content with |
| `flags` | no | Regex flags (default: `"gi"`) |
| `name` | no | A human-readable label for the rule (default: `"custom"`) |

Example:

```json
{
  "redactPatterns": [
    {
      "name": "internal-project-id",
      "pattern": "PROJ-[A-Z0-9]{6}",
      "replacement": "<REDACTED-PROJ>",
      "flags": "gi"
    },
    {
      "name": "custom-email",
      "pattern": "[a-z0-9._%+-]+@[a-z0-9.-]+\\.[a-z]{2,}",
      "replacement": "<REDACTED-EMAIL>"
    }
  ]
}
```

Custom patterns are loaded at startup via `sanitize.setCustomPatterns()` and
applied after all built-in patterns. They are used by both `redactText()` and
`redactWithDiff()`, so they also appear in the preview diff view.

### Redaction preview

The `preview` command (`npm run preview`) shows what will be archived
**without writing any files**. It displays new and changed sessions with
their redacted title, task type, and message count, so you can verify
redaction before committing to an archive run.

Use `preview --source <name>` to probe and preview a specific source
adapter (e.g., `--source claude-code`).

The `redactWithDiff()` function in `src/sanitize.js` returns both the
redacted result and a list of individual redactions that were applied,
enabling transparent diff views in the terminal and future tooling.

If you spot a real username or credential in a generated user-visible file,
that is a bug. Run `npm run verify` before sharing generated output.

## About `data/index.json`

`data/index.json` is a local fingerprint cache. It maps each source session file
to its last known fingerprint and generated task records, allowing incremental
runs to skip unchanged files.

The cache can contain original absolute paths as object keys for stable lookup.
It is gitignored and should not be shared.

## Sharing The Project

Before publishing or sharing the repository, run:

```bash
# Cross-platform, from the project root
npm run verify:fresh
npm pack --dry-run
git status --short --ignored --untracked-files=all
```

Generated personal files should appear as ignored, not as files to commit.

## Wiping Generated Outputs

If you want to remove local archive outputs, delete only the generated contents
inside these directories:

- `data/`
- `journal/`
- `reports/`
- `dist/`

Keep each directory's `.gitkeep` and README placeholder if you are preparing a
public repository.
